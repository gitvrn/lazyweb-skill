Systemic findings (verified directly against the live PRO API)

CONFIRMEDhigh Desktop search returns screenshotId: null

Every platform:desktop result has screenshotId: null (mobile results have it). So lazyweb_find_similar — which requires a numeric id — cannot be driven from desktop searches at all. Hit in T1, T4, T5, T6. The fields param does not repopulate it.

CONFIRMEDhigh company filter is case-sensitive (brand names silently return 0)

company:"Vercel" → 0 results; company:"vercel" → 1. The stored value is a lowercase crawl slug. Every skill tells the agent to pass brand-cased company names (e.g. --company "stripe"), which silently yields nothing. Hit in T4 (Vercel, OpenAI both 0).

CONFIRMEDhigh ab_test_research never returns embeddable experiment images

control.image_url / variant.image_url are null on every experiment. The ab-test & paywall skills instruct agents to embed those URLs — impossible. Agents had to re-source equivalent paywalls via lazyweb_search (T7, T8), adding work and a layer of approximation between the cited experiment and the shown pixels.

CONFIRMEDmed companyName is the crawl seed, not the page owner

Worst case T4: bucket segment shown as a Twilio WhatsApp page; microsoft-azure-quantum → Azure calculator; aws-trainium → AWS pricing; chatgpt → OpenAI API pricing. The brand must be re-derived from pageUrl. This is the single biggest driver of mislabeled / mismatched references (the weakest criterion).

CONFIRMEDmed visionDescription is null on find_similar + some newer captures

Baseline search DOES return visionDescription (~200–420 chars). But find_similar results and certain newer compare/ & uploaded_ captures return it empty. The skills' core gate is ‘caption only from visionDescription, else skip’ — so agents either dropped good refs or inferred captions, and the inferred ones were where false captions appeared (T2 BetterMe, T5 GitHub ‘trusted-by logos below CTA’ / ‘product video’ — both false).

CONFIRMEDmed high_design_bar: true is silently ignored on the public gateway

Accepted but echoed back false (T7). The corpus is therefore never taste-filtered even when the skill thinks it asked for it.

CONFIRMEDlow Shared pro token rate-limits under concurrency

MCP_RATE_LIMITED on get_flows / list_categories when ~8 agents shared one token (T9, T10). Retries recovered, but a single user running one skill won't see this; a team or a fan-out will.

CONTEXTlow _experiments corpus is mobile-subscription-centric

ab_test_research respects category, but the underlying corpus is overwhelmingly mobile-app paywall / subscription / onboarding tests. For the web-marketing briefs (T1 dev-email API, T4 usage-based pricing, T5 dev-tool hero) there is essentially no on-context experiment, so the A/B section is inherently a stretch — agents correctly labeled it caveated, hypothesis-grade evidence.

CRAFTmed Storage serves non-production captures (/figma/, !updating, !purchase-flow)

Some embedded references resolve to Figma design assets or transitional/loading states rather than shipped UI (T7 Headspace /figma/, T9 Zero /figma/, T6 Oura !updating). These pass as ‘real app screenshots’ but aren't.

What to change

In the skills (today)

  • Embed-count honesty: stop printing ‘22 Lazyweb references’ in the header — agents inflated it vs. the real <img> count in T2 and T6. Compute the badge from the DOM, or drop it.
  • Every skill should add an in-page jump-nav: the #1 navigation fix in 9 / 10 reports was ‘no id= anchors, no sticky TOC’ on 400–600-line reports.
  • Provenance contract should require deriving brand from pageUrl (not companyName) and tagging any /figma/ or ! path as ‘non-production’.
  • When visionDescription is null, the skills should say ‘read the image, then caption — and mark the caption as agent-observed’, never infer silently. The false captions all came from silent inference.
  • Quantitative claims (‘78% of onboarding flows show progress’, ‘~half of B2B dev pages’, ‘1.78% install-to-conversion’) need a named source or an explicit ‘estimate’ tag — they were the most common why/trust red flag.

In the backend / MCP (higher leverage)

  • Return screenshotId on desktop results (or accept siteId in find_similar).
  • Make the company filter case-insensitive / slug-normalize the input.
  • Either populate control/variant.image_url in ab_test_research or remove that instruction from the skills and document the search-resourcing workaround.
  • Honor high_design_bar on the public gateway, or reject it so the agent knows it didn't apply.
  • Backfill visionDescription for find_similar + newer capture buckets, and exclude /figma/ / transitional captures from default results.
